## 시나리오 1: 매일 아침기도 — 교차 그룹 기도 흐름 (P1 바트 신부 — 모바일)

### 상황
바트 신부는 매일 아침 6시에 성무일도 아침기도를 바친다. 오늘은 2주차 화요일, 연중 시기이다. 아침기도는 시편집의 해당 요일 섹션 + 연중 시기 고유문 + 찬미가를 **순서대로 교차 이동**하며 바쳐야 한다.

### 실제 기도 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 브라우저에서 앱 열기 | 마지막 읽기 위치 자동 복원 | ✅ 구현 |
| 2 | 사이드바 열기 → 시편집 2주차 화요일 | 아침기도 첫 섹션으로 이동 | ✅ 구현 |
| 3 | **시편집**: 아침기도 시편 1 스크롤하며 읽기 | 본문 표시, 브레드크럼 업데이트 | ✅ 구현 |
| 4 | 본문 중 "Магтуу: х. 879" 참조 발견 | 현재: plain text (클릭 불가) | ❌ F-XNAV 미구현 |
| 5 | **사이드바로 찬미가 이동**: 찬미가 → 해당 곡 찾기 | 사이드바 수동 탐색으로 이동 가능 | ⚠️ 수동만 가능 |
| 6 | 찬미가 부른 후 **사이드바로 시편집 복귀** | 이전 위치 기억 안 됨 — 다시 찾아야 함 | ⚠️ F-FLOW 미구현 |
| 7 | **시편집**: 아침기도 시편 2 이어 읽기 | 스크롤로 다음 섹션 | ✅ 구현 |
| 8 | **시편집**: 찬가 (Шад магтаал) 읽기 | 연속 스크롤 | ✅ 구현 |
| 9 | 본문 중 "시기별 고유문 참조" 발견 | 현재: plain text | ❌ F-XNAV 미구현 |
| 10 | **사이드바로 고유문 이동**: 연중 시기 → 해당 주일 | 수동 탐색 | ⚠️ 수동만 가능 |
| 11 | 고유문의 독서/응답구 읽기 | 본문 표시 | ✅ 구현 |
| 12 | **사이드바로 시편집 복귀** → 마침기도 | 이전 위치를 다시 찾아야 함 | ⚠️ F-FLOW 미구현 |
| 13 | 마침기도 읽기 → 기도 완료 | 위치 저장 | ✅ 구현 |

### 핵심 문제
- **교차 이동 3회** (시편집→찬미가→시편집→고유문→시편집)가 모두 수동 사이드바 탐색에 의존
- 이동할 때마다 **이전 읽기 위치가 유실**됨 — 돌아올 곳을 기억해야 함
- 본문 내 `х. 879` 참조가 클릭 불가

### 개선 기회
- **F-XNAV** (P0): 본문 내 `х. NNN` 참조를 클릭 가능한 링크로 변환 → 원클릭 이동
- **F-FLOW** (P1): 기도 흐름 가이드 — "이전 위치로 돌아가기" 버튼, 이동 이력 스택
- **F-CALENDAR** (P3): 전례력 연동으로 오늘 기도에 필요한 섹션 자동 수집

## 시나리오 2: 사순 시기 전례 준비 (P2 김 선교사 — 데스크탑)

### 상황
김 선교사는 다음 주일 미사 전례를 준비하며, 사순 시기 3주일 고유문을 확인하고자 한다.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 데스크탑 크롬에서 앱 열기 | SSR로 목차 즉시 표시, 사이드바 열림 | ✅ 구현 |
| 2 | 사이드바에서 "Цаг улирлын онцлог шинж" (시기별 고유문) 클릭 | 트리 펼침 → 6개 시즌 표시 | ✅ 구현 |
| 3 | "Дөчин хоногийн цаг улирал" (사순) 클릭 | 사순 하위 항목 펼침 (요일, 성주간 등) | ✅ 구현 |
| 4 | "ГУРАВ ДАХЬ НЯМ ГАРАГ" (3주일) 클릭 | 해당 주일 고유문 본문 표시 + 스크롤 | ✅ 구현 |
| 5 | 폰트 크기 A+ 클릭 (2회) | 18px → 22px로 확대 | ✅ 구현 |
| 6 | 텍스트 복사하여 전례 준비 문서에 붙여넣기 | 일반 텍스트 선택/복사 가능 | ✅ 구현 |

### 개선 기회
- **본문 검색** (P0): "ГУРАВ" 검색으로 3주일을 직접 찾기 (현재 제목만 검색)
- **전례 텍스트 구조 마크업** (P0): 시편, 독서, 응답구가 시각적으로 구분되면 준비 효율 상승
- **인쇄 최적화** (P3): 준비 문서용 인쇄 레이아웃

## 시나리오 3: 야간 기도 (P3 에르덴 수녀 — 태블릿)

### 상황
에르덴 수녀는 밤 9시 침실에서 끝기도(저녁 마침 기도)를 바친다. 어두운 환경에서 눈의 피로를 줄이고 싶다.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 태블릿에서 앱 열기 | 이전에 설정한 다크 모드 즉시 적용 (FOUC 없음) | ✅ 구현 |
| 2 | 마지막 읽기 위치에서 이어서 표시 | 저녁기도 섹션 자동 복원 | ✅ 구현 |
| 3 | 폰트 크기 확인 | 이전 설정 26px 유지 | ✅ 구현 |
| 4 | 저녁기도 본문 읽기 | 다크 모드: stone-950 배경 + stone-100 텍스트 | ✅ 구현 |
| 5 | 스크롤하며 시편, 찬미가 순서대로 기도 | 부드러운 스크롤, 활성 섹션 추적 | ✅ 구현 |
| 6 | 기도 후 앱 닫기 | 위치 저장 | ✅ 구현 |

### 개선 기회
- **시스템 다크모드 연동** (P1): iPad 시스템 설정 따라 자동 전환 (현재 수동 토글만)
- **루브릭 구분** (P0): 사제 지시문은 어두운 빨간색으로 구분하면 기도 흐름 파악 용이
- **자동 스크롤/자동 넘기기** (P3): 일정 시간 후 다음 절로 자동 이동

## 시나리오 4: 찬미가 검색 (P4 볼드 평신도 — 모바일)

### 상황
볼드는 성당 미사 중 신부님이 "42번 찬미가"를 부르자고 했다. 빠르게 가사를 찾아야 한다.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 스마트폰에서 앱 열기 | 마지막 위치 복원 (시편집 영역) | ✅ 구현 |
| 2 | 사이드바 열기 | 모바일 오버레이 | ✅ 구현 |
| 3 | 검색창에 "42" 입력 | 제목에 "42"가 포함된 결과 표시 | ✅ 구현 |
| 4 | "42. Есүс хамгийн нандин нэр юм аа" 터치 | 해당 찬미가로 이동, 사이드바 자동 닫힘 | ✅ 구현 |
| 5 | 가사를 보며 따라 부름 | 본문 표시, 서체 가독성 확보 | ✅ 구현 |

### 개선 기회
- **본문 검색** (P0): "Есүс" (예수)를 검색하면 가사 본문에서도 찾기
- **찬미가 번호 직접 이동** (P1): "42" 입력 시 찬미가 42번으로 즉시 이동 (현재 제목 매칭만)
- **폰트 더 크게** (P1): 성당에서 먼 거리에서도 보이도록 40px+ 지원

## 시나리오 5: 처음 방문 (모든 페르소나)

### 상황
처음으로 앱 URL을 받아 접속하는 사용자.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | URL 접속 | 목차 사이드바 + 스켈레톤 로딩 즉시 표시 | ✅ 구현 |
| 2 | 첫 콘텐츠 그룹 자동 로드 | week1 본문 표시 (1주차 토요일 저녁기도부터) | ✅ 구현 |
| 3 | 사이드바 트리 탐색 | 1~2depth 기본 펼침, 나머지는 클릭으로 펼침 | ✅ 구현 |
| 4 | 아무 항목 클릭 | 해당 섹션으로 스크롤 + 콘텐츠 로드 | ✅ 구현 |
| 5 | 다크모드 토글 | 즉시 전환, 설정 저장 | ✅ 구현 |

### 개선 기회
- **온보딩 가이드** (P1): 처음 사용자에게 간단한 사용법 툴팁 (1회성)
- **빈 상태 안내** (P1): "좌측 목차에서 기도문을 선택하세요" 같은 안내 메시지
- **홈 화면 추가 안내** (P2): PWA 설치 유도 배너

## 시나리오 6: 특정 시편 찾기 (P2 김 선교사 — 데스크탑)

### 상황
김 선교사가 "시편 141편"이 어느 기도시간에 배정되어 있는지 찾고 싶다.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 검색창에 "141" 입력 | "Дуулал 141"이 제목에 포함된 leaf 표시 | ✅ 구현 |
| 2 | 검색 결과 중 원하는 항목 클릭 | 해당 섹션으로 이동 | ✅ 구현 |
| 3 | 브레드크럼으로 위치 확인 | "4주 시편집 / 1주차 / 토요일 / 저녁기도" | ✅ 구현 |

### 개선 기회
- **시편 번호별 인덱스** (P1): 전용 시편 인덱스 페이지에서 번호별 빠른 접근
- **교차 참조** (P3): "이 시편은 다음 기도시간에도 사용됩니다" 링크

## 시나리오 7: 인터넷 불안정 환경 (P1 바트 신부 — 모바일)

### 상황
몽골 시골 성당에서 인터넷이 느리거나 끊긴다.

### 흐름 (현재)

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 앱 열기 | 목차는 SSR로 표시, 본문은 로딩 스켈레톤 | ✅ 구현 |
| 2 | 콘텐츠 로드 시도 | 느린 네트워크 — 스켈레톤이 오래 지속 | ⚠️ 개선 필요 |
| 3 | 네트워크 끊김 | 콘텐츠 로드 실패, 스켈레톤 영구 표시 | ❌ 미해결 |

### 개선 기회
- **오프라인 캐싱** (P2): Service Worker로 전체 780KB 콘텐츠 캐싱 → 오프라인 완전 지원
- **로드 실패 재시도 UI** (P1): "다시 시도" 버튼 표시
- **캐시 상태 표시** (P2): "오프라인 모드" 인디케이터

## 시나리오 8: 전례 시기 전환 (P3 에르덴 수녀 — 태블릿)

### 상황
대림 시기가 끝나고 성탄 시기가 시작된다. 고유문이 바뀌어야 한다.

### 흐름

| 단계 | 행동 | 시스템 반응 | 상태 |
|------|------|-----------|------|
| 1 | 사이드바에서 "시기별 고유문" 펼치기 | 6개 시즌 표시 | ✅ 구현 |
| 2 | "대림" 접기, "성탄" 펼치기 | 성탄 하위 항목 표시 | ✅ 구현 |
| 3 | "ЭЗЭНИЙ МЭНДЭЛСЭН ӨДӨР" (성탄 대축일) 클릭 | 성탄 고유문 표시 | ✅ 구현 |
| 4 | 시편집 1주차와 교차 사용 | 사이드바에서 시편집 → 1주차 이동 | ✅ 구현 |

### 개선 기회
- **전례력 자동 선택** (P3): 오늘 날짜로 현재 시기와 해당 고유문 자동 제안
- **시기별 테마 색상** (P1): 성탄 시기에는 금색/흰색 악센트

---

## 시나리오 9: 완전한 저녁기도 — 교차 그룹 전체 흐름 (P3 에르덴 수녀 — 태블릿)

### 상황
에르덴 수녀는 1주차 일요일 저녁기도를 바친다 (대림 시기). 저녁기도는 시편집 + 대림 고유문 + 찬미가를 순서대로 교차하며 읽어야 한다.

### 실제 기도 흐름

| 단계 | 출처 | 행동 | 시스템 반응 | 상태 |
|------|------|------|-----------|------|
| 1 | - | 앱 열기, 다크 모드 적용 | FOUC 없이 즉시 다크 | ✅ 구현 |
| 2 | **시편집** | 사이드바 → 1주차 → 일요일 → 저녁기도 섹션 1 | 시편 본문 표시 | ✅ 구현 |
| 3 | **시편집** | 시편 스크롤하며 읽기 | 연속 스크롤 | ✅ 구현 |
| 4 | (참조) | 본문에 "Магтуу: х. 874" 발견 | plain text, 클릭 불가 | ❌ F-XNAV |
| 5 | **찬미가** | 사이드바 → 찬미가 → 해당 곡 수동 탐색 | 그룹 전환, 찬미가 로드 | ⚠️ 수동 |
| 6 | **찬미가** | 찬미가 부름 | 본문 표시 | ✅ 구현 |
| 7 | **시편집** | 사이드바 → 시편집 1주차 일요일로 **복귀** | 이전 위치 유실, 다시 찾아야 함 | ⚠️ F-FLOW |
| 8 | **시편집** | 저녁기도 섹션 2 (찬가) 스크롤 | 연속 스크롤 | ✅ 구현 |
| 9 | (참조) | 독서 부분에서 "시기별 고유문" 참조 | plain text | ❌ F-XNAV |
| 10 | **고유문** | 사이드바 → 고유문 → 대림 → 해당 주일 | 그룹 전환, 고유문 로드 | ⚠️ 수동 |
| 11 | **고유문** | 대림 시기 독서/응답구 읽기 | 본문 표시 | ✅ 구현 |
| 12 | **시편집** | 사이드바 → 시편집으로 **재복귀** | 이전 위치 다시 유실 | ⚠️ F-FLOW |
| 13 | **시편집** | 마침기도 읽기 → 기도 완료 | 위치 저장 | ✅ 구현 |

### 핵심 문제
- 1회 기도에 **그룹 간 이동 4회** (시편집→찬미가→시편집→고유문→시편집)
- 매번 **사이드바 수동 탐색** 필요 (3~5번 클릭)
- **복귀 시 이전 위치 유실** — 어디까지 읽었는지 기억해야 함

## 우선순위 요약

시나리오 분석 기반 가장 영향력 큰 미구현 기능:

1. **교차 그룹 네비게이션 F-XNAV** — S1, S9에 영향. 기도 중 시편집↔고유문↔찬미가 이동의 핵심
2. **기도 흐름 가이드 F-FLOW** — S1, S9에 영향. 이동 후 이전 위치 복귀
3. **오프라인 지원 (PWA)** — S1, S4, S7에 영향. 몽골 시골 성당 환경에서 필수
4. **본문 검색** — S2, S4, S6에 영향. 436개 섹션에서 원하는 텍스트를 찾는 핵심 기능
5. **전례력 자동 선택** — S1, S8, S9에 영향. 매일 반복 사용자의 탐색 시간 대폭 절감
